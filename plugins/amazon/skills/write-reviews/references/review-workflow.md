# Review Workflow

## Entry Points

- Product pages may expose `Write a customer review`.
- Order details may expose review actions for purchased items.
- The review form may require selecting a star rating, title, body text, media choices, and optional feature ratings.

## Drafting

- Keep the review specific to the user's experience, product use case, and actual trade-offs.
- Avoid unsupported claims, private information, seller disputes unrelated to product performance, and medical/legal guarantees.
- If the user gives sparse input, produce a modest review that does not invent usage details.

## Submission

- Fill the form only with content the user approved or asked you to draft.
- Stop before the final submit/post action and ask for explicit approval if approval has not already been given for the final visible content.
- After submitting, verify a thank-you, confirmation, or submitted-review state before reporting success.

## Recovery

- If Amazon asks for sign-in or reauthentication, pause and ask the user to complete it in the browser.
- If the product is not reviewable, report the visible blocker.
- If the submitted state is unclear, say that the form was completed but submission could not be verified.
